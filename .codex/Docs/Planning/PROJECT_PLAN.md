# 프로젝트 상태 기록

> 이 문서는 완성형 설계 문서가 아니다.  
> 현재 확정된 기획과 확인 필요 항목을 빠르게 보기 위한 작업 문서다.

## 운영 규칙

- 초반부터 전체 구조를 완벽하게 설계하지 않는다.
- 개발하면서 확인된 내용만 추가한다.
- 현재 존재하지 않는 구현물은 실제 존재하는 것처럼 쓰지 않는다.
- 씬 구조, 저장 시스템, 전체 UI 구조, 세부 게임 루프는 구현하면서 조정될 수 있다.
- 실제 구현하면서 자연스럽게 결정될 내용은 `확인 필요`로 남긴다.
- 작업 항목은 `.codex/Docs/TODO.md` 한 곳에서만 관리한다.
- 사용자가 기획 내용을 직접 추가하면 그 내용을 기준으로 다른 하네스 문서도 보강한다.

## 게임 개요

- 장르: 종합 미니게임 / 싱글플레이
- 플랫폼: PC
- 한 줄 소개:
  - 제한 시간 안에 다양한 야생동물과 관련된 미니게임을 최대한 많이 클리어하며 점수를 기록하는 게임

## 핵심 컨셉

- 코어 루프:
  - 게임 시작
  - 랜덤하게 나온 미니게임 선택
  - 제한 시간 동안 미니게임 플레이
  - 다음 미니게임 반복
  - 결과 확인
- 게임 목표:
  - 제한 시간 내에 미니게임을 최대한 많이 클리어하여 높은 점수 달성
- 차별점:
  - 야생동물 컨셉
  - 병맛 분위기
  - 랜덤 미니게임 진행
  - 빠른 선택 및 진행 구조

## 게임플레이

- 기본 조작:
  - 이동: WASD
  - 점프: Space
  - 물기: 좌클릭
- 기본 조작은 미니게임에 따라 변동될 수 있다.
- 핵심 메커니즘:
  - 여러 미니게임이 랜덤으로 등장
  - 스테이지 선택 시스템
  - 제한 시간 시스템
  - 선택 시간에도 타이머 진행
- 게임 흐름:
  - 메인 화면
  - 게임 시작
  - 연습모드 / 게임모드 선택
  - 스테이지 후보 표시
  - 플레이어 선택
  - 미니게임 진행
  - 반복
  - 결과 화면 이동
- 난이도 설계:
  - 각 미니게임마다 난이도가 다르다.
  - 난이도에 맞게 얻을 수 있는 점수도 다르다.

## 콘텐츠

- 스테이지 / 맵 구성:
  - 여러 미니게임 스테이지 랜덤 진행
