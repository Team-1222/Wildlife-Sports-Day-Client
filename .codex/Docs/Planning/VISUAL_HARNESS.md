# 화면 / UX 하네스

> 시각 방향은 `PROJECT_PLAN.md`의 확정 항목을 따른다. 세부 스타일은 구현하면서 보강한다.

## 시각 방향

- 장르: 종합 미니게임 / 싱글플레이
- 핵심 분위기: 병맛
- 아트 스타일: 확인 필요
- 픽셀아트 여부: 확인 필요
- 색상 방향: 확인 필요
- 포인트 컬러: 확인 필요
- 폰트 방향: 확인 필요
- UI 밀도: 확인 필요

## UI 방향

- 확정된 병맛 분위기는 유지하되, 세부 UI 스타일은 아직 강제하지 않는다.
- 화면 정보는 읽기 쉬운 크기와 대비를 우선한다.
- 버튼, 입력, 선택 상태는 즉시 구분되어야 한다.
- 임시 UI라도 텍스트 넘침과 Canvas 깨짐은 방치하지 않는다.
- 새 UI는 실제 구현 또는 사용자 결정으로 화면 구조가 정해진 뒤 정리한다.

## UX 방향

- 기본 조작은 WASD, Space, 좌클릭이지만 미니게임별 변동 가능성을 열어둔다.
- 플레이 흐름, 메뉴 흐름, 결과 흐름은 실제 결정이 생긴 뒤 보강한다.
- 화면 전환은 사용자가 현재 상태를 이해할 수 있게 유지한다.
- 테스트용 UI와 실제 UI를 구분한다.

## 확인된 화면 / 씬 기준

- `SampleScene`: Unity 기본 샘플 씬

## UI 가독성 체크리스트

- [ ] 텍스트가 배경과 충분히 대비되는가
- [ ] 버튼/패널 안에서 텍스트가 넘치지 않는가
- [ ] 긴 한글 문자열이 잘리지 않는가
- [ ] 선택/hover/disabled 상태가 구분되는가
- [ ] 중요한 정보가 화면 가장자리나 노치 영역에 붙지 않는가

## Canvas / 해상도 체크리스트

- [ ] 기준 해상도에서 Canvas가 깨지지 않는가
- [ ] 16:9 해상도에서 UI가 겹치지 않는가
- [ ] 창 크기 변경 시 주요 UI가 화면 밖으로 나가지 않는가
- [ ] 앵커와 피벗이 의도대로 설정되어 있는가
- [ ] Safe Area 대응 필요 여부를 확인했는가

## 아트 검수 체크리스트

- [ ] 스프라이트 해상도와 필터링이 의도와 맞는가
- [ ] 폰트, 색상, 외곽선, 배경 대비가 읽기 쉬운가
- [ ] 움직이는 요소가 조작이나 판정을 가리지 않는가
- [ ] 임시 리소스와 최종 리소스가 구분되는가
- [ ] Unity 씬, 프리팹, 메타 파일 참조가 깨지지 않았는가
